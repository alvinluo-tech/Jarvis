# Jarvis Personal Command Center — 产品需求文档 (PRD)

## 1. 产品概述

Jarvis 是一个桌面端个人指令中心，将多个自建 Web 应用（Todo、阅读清单、健身、日历）统一在 AI 驱动的界面之下。它不是通用聊天机器人，而是「个人数据中心 + 语音操作入口 + 自建 Web 应用的统一控制层」。

MVP 阶段聚焦三个模块：Todo 管理、阅读清单、每日/每周回顾，仅支持文本交互。

## 2. 用户画像

当前阶段唯一的用户是开发者本人。典型使用场景：

- 开始工作时，问 Jarvis「今天有什么任务？」快速获取当日待办
- 阅读文章时，说「把这篇文章加到阅读清单」
- 周末复盘时，要求「总结本周完成情况，分析模式，建议改进」
- 日常通过自然语言管理任务优先级，无需手动操作 UI

## 3. 功能需求

### 模块 1：Todo 管理

| ID | 功能 | 描述 |
|---|---|---|
| T-01 | 创建任务 | 通过自然语言创建任务，AI 解析标题、优先级、截止日期、标签 |
| T-02 | 查询任务 | 支持按日期、优先级、标签、状态筛选 |
| T-03 | 更新任务 | 修改状态、优先级、截止日期 |
| T-04 | 删除任务 | 软删除，保留历史 |
| T-05 | 优先级管理 | AI 辅助排序，基于截止日期和重要性 |
| T-06 | 今日视图 | 获取今日到期 + 高优先级未完成任务 |

### 模块 2：阅读清单

| ID | 功能 | 描述 |
|---|---|---|
| R-01 | 添加文章 | 通过 URL 或标题添加，AI 自动提取元信息 |
| R-02 | 阅读进度 | 记录已读/在读/未读状态 |
| R-03 | 阅读统计 | 按时间维度统计阅读量 |
| R-04 | 推荐下一读 | AI 基于阅读历史和兴趣推荐 |
| R-05 | 文章评分 | 1-5 星评分 + 简短笔记 |
| R-06 | 分类管理 | 按主题/来源分类 |

### 模块 3：每日/每周回顾

| ID | 功能 | 描述 |
|---|---|---|
| V-01 | 每日总结 | 当日任务完成率、阅读量 |
| V-02 | 每周总结 | 周维度统计数据、趋势分析 |
| V-03 | 模式识别 | AI 识别拖延模式、高效时段、习惯趋势 |
| V-04 | 改进建议 | 基于数据给出具体可行的改进建议 |
| V-05 | 历史回顾 | 查看历史总结记录 |

## 4. 非功能需求

### 性能
- AI 响应时间 < 3 秒（不含模型推理时间）
- 本地 SQLite 查询 < 50ms
- Supabase 云同步延迟 < 2 秒
- 应用启动时间 < 2 秒

### 安全
- Supabase RLS 策略确保数据隔离
- API Key 通过环境变量管理，不硬编码
- 本地 SQLite 数据加密存储
- 所有用户输入在边界处验证

### 可维护性
- 单文件 < 800 行，单函数 < 50 行
- 模块间低耦合，通过 Tool 接口通信
- 不可变数据模式
- 80%+ 测试覆盖率

## 5. 成功指标

- 能通过自然语言完成 Todo 的增删改查全流程
- 每日回顾总结准确反映当日工作状态
- 每周回顾能识别出至少 1 个有意义的行为模式
- AI 工具调用成功率 > 95%
- 开发者日常使用后，任务完成率可量化追踪

## 6. MVP 范围外

- 语音输入/输出（Phase 2）
- 健身模块（Phase 2）
- 日历模块（Phase 2）
- 多用户支持
- 移动端
- 离线 AI 推理
- 自定义 UI 主题

## 7. 技术栈

- **Desktop**: Tauri 2.0 + React + Vite + Tailwind CSS + shadcn/ui
- **Backend/Agent**: Node.js/TypeScript daemon
- **Cloud DB**: Supabase (PostgreSQL)
- **Local DB**: SQLite + Drizzle ORM
- **AI**: MiMo model (Xiaomi) via API + tool calling pattern
- **Protocol**: MCP (Model Context Protocol) for tool integration
